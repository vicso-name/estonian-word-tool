# DRILL ENGINE — Алгоритм и спецификация
# ══════════════════════════════════════════════════════════════

## 1. АРХИТЕКТУРА СЕССИИ

Сессия дрилла состоит из 3 раундов. Каждый раунд — своя когнитивная нагрузка:

  Раунд 1 — УЗНАВАНИЕ      (difficulty=1)
    Задача: активировать слово из пассивной памяти
    Тип:    RU → ET (базовая форма)
    Режим:  свободный ввод + подсказка доступна

  Раунд 2 — ФОРМЫ          (difficulty=2)
    Задача: воспроизвести конкретную грамматическую форму
    Тип:    форма по заданию (osastav, pres_3sg и т.д.)
    Режим:  свободный ввод, без подсказки

  Раунд 3 — ПРОДУКЦИЯ      (difficulty=3)
    Задача: использовать слово в контексте
    Тип:    сложные формы + AI-контекст
    Режим:  свободный ввод, AI проверяет


## 2. ШАБЛОНЫ УПРАЖНЕНИЙ

Каждый шаблон строго определён:

  id:             уникальный идентификатор
  pos:            список допустимых частей речи [NOUN, VERB, ADJ, ADV]
  requires:       поля которые ДОЛЖНЫ быть непустыми в БД (иначе шаблон пропускается)
  difficulty:     1 | 2 | 3 (раунд)
  prompt_tpl:     шаблон вопроса с плейсхолдерами {lemma}, {translation}, {form}
  answer_field:   поле БД которое является правильным ответом
  hint_tpl:       подсказка (показывается после 2й неудачной попытки)
  grammar_note:   объяснение правила с плейсхолдерами форм


## 3. ВАЛИДАЦИЯ ОТВЕТА (строгий порядок)

  Шаг 1 — Нормализация
    - strip() + lowercase()
    - убрать лишние пробелы
    - убрать точку в конце

  Шаг 2 — Точное совпадение
    normalize(ответ) == normalize(ожидаемый)
    → CORRECT, без AI

  Шаг 3 — Мягкое совпадение (опечатки)
    Levenshtein distance <= 1 И длина слова >= 4
    → ALMOST (показать правильный ответ, засчитать как ошибку но объяснить)

  Шаг 4 — AI валидация (только если шаги 2-3 не сработали)
    Вызывать ТОЛЬКО для difficulty=3 или если ответ длиннее одного слова
    Промпт: строгий, с указанием ожидаемой формы и всех форм слова
    AI может вернуть: CORRECT | WRONG | PARTIAL
    → Если PARTIAL: засчитать ошибку, показать объяснение

  Шаг 5 — WRONG
    Показать правильный ответ + grammar_note
    Слово добавляется в очередь повторения (в конец текущего раунда)


## 4. ПОСТРОЕНИЕ СЕССИИ

  Вход: список слов из БД (list_words)

  1. Для каждого слова определить применимые шаблоны:
       applicable = [t for t in TEMPLATES
                     if word.pos in t.pos
                     and all(word.get(f) for f in t.requires)]

  2. Если у слова 0 применимых шаблонов → пропустить слово

  3. Для Раунда 1: взять ровно 1 шаблон difficulty=1 на слово
     Для Раунда 2: взять все шаблоны difficulty=2 на слово
     Для Раунда 3: взять шаблоны difficulty=3 (если есть)

  4. Внутри раунда — перемешать (random.shuffle)

  5. Итоговая очередь: Round1 + Round2 + Round3
     Максимум: 40 упражнений на сессию (срезать Round3 если больше)

  6. Сохранить в session['drill']:
       {
         list_id, grammar_topic, exercises: [...],
         current: 0, results: [], started_at
       }


## 5. СТРУКТУРА УПРАЖНЕНИЯ (в session)

  {
    word_id:      int,
    template_id:  str,
    lemma:        str,
    prompt:       str,       # отрендеренный вопрос
    answer:       str,       # ожидаемый ответ (нормализованный)
    answer_field: str,       # для подсветки в таблице форм
    forms:        dict,      # все формы слова (для AI контекста)
    grammar_note: str,       # отрендеренная подсказка с реальными формами
    difficulty:   int,
    round:        int,
    attempts:     int,       # начинается с 0
  }


## 6. ГРАММАТИЧЕСКАЯ ТЕМА

  Пользователь указывает тему в начале сессии (опционально).
  Примеры: "партитив", "отрицание", "порядок слов", "согласование прил."

  Влияет на:
    1. Приоритет шаблонов: шаблоны связанные с темой идут первыми
    2. AI промпт: тема добавляется в системный промпт
    3. Объяснения: AI фокусирует объяснение на теме

  Маппинг тема → приоритетные шаблоны:
    "партитив"     → noun_osastav, adj_osastav, verb_da_inf
    "спряжение"    → verb_pres_3sg, verb_past_3sg
    "инфинитив"    → verb_ma_inf, verb_da_inf
    "падежи"       → noun_omastav, noun_osastav


## 7. AI ПРОМПТ (строгий шаблон)

  System:
    Ты эксперт по эстонской грамматике. Отвечай ТОЛЬКО в JSON.
    Грамматическая тема урока: {grammar_topic or "общая лексика"}.

  User:
    Слово: {lemma} ({translation})
    Все формы: {json.dumps(forms)}
    Задание: {prompt}
    Ожидаемый ответ: {answer}
    Ответ ученика: {student_answer}

    Верни JSON:
    {
      "verdict": "CORRECT" | "PARTIAL" | "WRONG",
      "is_acceptable": bool,
      "explanation": "1-2 предложения на русском, фокус на {grammar_topic}",
      "correct_form": "{answer}"
    }

    Правила:
    - CORRECT если ответ правильный (возможны мелкие опечатки длиной 1)
    - PARTIAL если смысл верный но форма неточная
    - WRONG если неправильная форма или не то слово
    - explanation максимум 2 предложения
    - НЕ пиши ничего кроме JSON


## 8. РЕЗУЛЬТАТЫ СЕССИИ

  После последнего упражнения → страница результатов:

  {
    total:         int,
    correct:       int,
    partial:       int,
    wrong:         int,
    accuracy:      float,  # correct / total
    hard_words:    [...],  # слова с 2+ ошибками
    duration_sec:  int,
    by_round:      { 1: {...}, 2: {...}, 3: {...} }
  }

  hard_words автоматически предлагаются для следующей сессии
